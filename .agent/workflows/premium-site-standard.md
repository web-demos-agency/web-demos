# 🚀 PREMIUM SITE GENERATION STANDARD v2.0 (The Fix)

Этот воркфлоу заменяет размытые понятия "Wow-effect" на конкретные инструменты и принципы для создания конверсионных, дорогих и масштабируемых сайтов.

## 🎨 1. Дизайн-система (Aesthetics & UX)
- **Сетка (Grid):** Использовать **Bento Grid** для секций преимуществ и галерей. Это современно, аккуратно и отлично смотрится на мобилках.
- **Типографика:** Минимум 2 шрифта. Заголовочный (Display/Serif/Bold Sans) и основной (Inter/Outfit/Geist). Иерархия размеров: h1 (64px+), h2 (48px+), p (18px+).
- **Цвета:** Использовать правило 60-30-10. Основной фон (Dark/Light), акцентный (Brand), вспомогательный. Никаких "чистых" #000 или #FFF. Только глубокие оттенки (напр. `#0a0b10`).
- **Градижеты и Тени:** Использовать **Soft Shadows** (многослойные тени) вместо резких рамок. Градиенты — только Mesh или Subtle Linear.
- **Анимации:** Scroll-driven animations через Intersection Observer. Плавные появления, а не дерганые переходы.

## 🎯 2. Маркетинг и Конверсия (CRO)
Сайт должен продавать, а не просто висеть в сети.
- **Структура Hero:** 
  1. Сильный оффер (Headline).
  2. Подзаголовок с выгодой (Subheadline).
  3. Основной CTA (Кнопка с контрастным цветом).
  4. Социальное доказательство (напр. "TrustScore 4.9/5").
- **Блоки доверия:** Реальные отзывы, сертификаты, логотипы партнеров.
- **Секция боли и решения (PMPHS):** Обозначить проблему клиента (напр. "Течет кран?") и предложить мгновенное решение.
- **FAQ:** Обязательный блок для снятия возражений и SEO.

## 🛠 3. Техническое SEO и Производительность
- **Semantic HTML:** Использовать `<header>`, `<main>`, `<section>`, `<footer>`, `<aside>`.
- **Structured Data (JSON-LD):** Обязательно внедрять разметку LocalBusiness для каждого сайта.
- **Performance:** 
  - Изображения в WebP.
  - Ленивая загрузка (`loading="lazy"`).
  - Критический CSS инлайном, остальное — в файл.
- **Accessibility:** Контрастность текста по WCAG, атрибуты `aria-label` для иконок без текста.

## 🖼 4. Контент и Медиа
- **Генерация:** Все изображения генерировать через `generate_image` с промптами, описывающими профессиональное освещение и фотореализм.
- **Иконки:** Использовать SVG (напр. Lucide или Phosphor), а не шрифты.

## 📄 5. Документация и Питч
- **Формат:** Для каждого лида создавать `pitch.md` (с форматированием) и ОБЯЗАТЕЛЬНО `pitch.txt` (чистый текст без Markdown для удобства копирования).
- **Подпись:** В качестве подписи по умолчанию использовать "Designer".
- **Base URL:** Использовать `https://web-demos-1.pages.dev/` для генерации ссылок на лендинги. Ссылка должна иметь формат `https://web-demos-1.pages.dev/[folder-name]/`.
